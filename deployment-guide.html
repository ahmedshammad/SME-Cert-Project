VM1 Only — Generate Crypto & Secrets

This is the primary node — do all of this only on VM1

3.1 — Generate crypto material & channel artifacts VM1 only

cd ~/bc_application/blockchain/network
chmod +x scripts/*.sh

# Generate certificates for all 4 orgs
./scripts/generate_crypto.sh

# Generate channel genesis block
./scripts/generate_artifacts.sh

# Verify
ls crypto-config/peerOrganizations/   # should show 4 org folders
ls channel-artifacts/                  # should show certificates.block

3.2 — Generate TLS certificate for Nginx VM1 only

mkdir -p ~/bc_application/infra/nginx/tls
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout ~/bc_application/infra/nginx/tls/server.key \
  -out  ~/bc_application/infra/nginx/tls/server.crt \
  -subj "/C=EG/ST=Cairo/L=Cairo/O=SME Platform/CN=$(curl -s ifconfig.me)"

3.3 — Generate production secrets & update .env VM1 only

cd ~/bc_application/infra/compose
VM_IP=$(curl -s ifconfig.me)

# Generate random secrets
JWT=$(openssl rand -base64 48)
MKEY=$(openssl rand -hex 32)
PG=$(openssl rand -base64 18 | tr -dc 'a-zA-Z0-9' | head -c 20)
GF=$(openssl rand -base64 12)

# Inject into .env
sed -i "s|sme-cert-jwt-secret-change-in-production-2024|$JWT|" .env
sed -i "s|0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef|$MKEY|" .env
sed -i "s|POSTGRES_PASSWORD=smepassword|POSTGRES_PASSWORD=$PG|" .env
sed -i "s|changeme_grafana|$GF|" .env
sed -i "s|CORS_ORIGIN=.*|CORS_ORIGIN=http://$VM_IP|" .env
sed -i "s|APP_URL=.*|APP_URL=http://$VM_IP|" .env
sed -i "s|SWAGGER_ENABLED=true|SWAGGER_ENABLED=false|" .env

# Remove empty SMTP lines (causes validation error if blank)
sed -i '/^SMTP_HOST=/d; /^SMTP_PORT=/d; /^SMTP_USER=/d; /^SMTP_PASS=/d' .env

# Add SMTP placeholder (replace with real credentials later)
cat >> .env << 'EOF'
SMTP_HOST=localhost
SMTP_PORT=25
SMTP_USER=noreply@localhost
SMTP_PASS=disabled
EOF

# Confirm
grep -E "POSTGRES_PASSWORD|CORS_ORIGIN|APP_URL|SWAGGER|JWT_SECRET" .env
echo "VM1 public IP: $VM_IP"
! Save the generated POSTGRES_PASSWORD value — you will need it when setting up the PostgreSQL replica on VM2.

Common API startup errors & fixes

ErrorCauseFix
SMTP_PASS is not allowed to be empty Empty string passed for SMTP vars Remove all SMTP_ lines from .env and add placeholder values
SMTP_HOST is not allowed to be empty Same — empty string vs missing sed -i '/^SMTP_HOST=/d; /^SMTP_PORT=/d; /^SMTP_USER=/d; /^SMTP_PASS=/d' .env then add placeholders
container is not running API crashed on startup Run docker logs sme-cert-api to see the actual error
fabric_network was not created for project Warning — not an error Safe to ignore. Fabric network is managed by the Fabric compose file.

9.1 — Sync wallets to VM2 Run on VM1

# One-time initial copy
scp -r ~/bc_application/wallets/ opc@<VM2_IP>:~/bc_application/wallets/
scp -r ~/bc_application/blockchain/network/connection-profiles/ \
  opc@<VM2_IP>:~/bc_application/blockchain/network/connection-profiles/
scp ~/bc_application/infra/nginx/tls/server.key \
    ~/bc_application/infra/nginx/tls/server.crt \
    opc@<VM2_IP>:~/bc_application/infra/nginx/tls/

# Auto-sync every 5 minutes via cron
(crontab -l 2>/dev/null; echo "*/5 * * * * rsync -avz ~/bc_application/wallets/ opc@<VM2_IP>:~/bc_application/wallets/ >> /tmp/wallet-sync.log 2>&1") | crontab -

9.2 — Copy .env to VM2 and update its IP Run on VM1

scp ~/bc_application/infra/compose/.env opc@<VM2_IP>:~/bc_application/infra/compose/.env

# On VM2 — update CORS_ORIGIN and APP_URL to VM2's IP
ssh opc@<VM2_IP> "
  VM2_IP=\$(curl -s ifconfig.me)
  sed -i \"s|CORS_ORIGIN=.*|CORS_ORIGIN=http://\$VM2_IP|\" ~/bc_application/infra/compose/.env
  sed -i \"s|APP_URL=.*|APP_URL=http://\$VM2_IP|\" ~/bc_application/infra/compose/.env
"

9.3 — Start app stack on VM2 Run on VM2

cd ~/bc_application/infra/compose

# It shares the same PostgreSQL via VM1's IP — or use a local replica (see 9.4)

9.4 — OCI Load Balancer (routes traffic to VM1 or VM2)

  1. Networking → Load Balancers → Create Load Balancer
    Shape: Flexible, 10 Mbps minimum
  2. Listeners: HTTP on port 80
  3. Backend Set: Add VM1 and VM2 on port 80
    Health check: HTTP, path /api/health, port 3000
  4. Result: LB sends traffic to VM1. If VM1 health check fails, it automatically routes to VM2.
✓ Update your DNS record to point to the Load Balancer IP instead of VM1 directly.
